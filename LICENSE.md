## 🛠️ Troubleshooting Common Issues

- ❌ **Krnl Executor fails to launch on Windows**
  The application requires `.NET Framework 4.7.2` or newer. Open `Settings > Apps > Optional Features` and verify .NET is installed. Reinstalling the runtime from Microsoft usually resolves this.
  **Solution:** Download and install the latest `.NET` runtime, then relaunch Krnl Executor.

- ❌ **Injection button stays grayed out**
  This typically means Roblox has not fully loaded into memory yet. Krnl Executor waits for the game client process before enabling injection.
  **Solution:** Wait until you are fully loaded into a Roblox game, then click `Inject` again.

- ❌ **Antivirus deletes Krnl Executor files**
  Script executors use techniques that trigger heuristic detections in most antivirus programs. This is a well-known false positive.
  **Solution:** Add the Krnl Executor installation folder (e.g., `C:\Program Files\KrnlExecutor`) to your antivirus exclusion list.

- ❌ **Krnl Executor 2026 Apk crashes on launch (Android)**
  Older Android WebView versions can cause instability with the embedded editor interface.
  **Solution:** Update Android System WebView and Chrome from the Play Store, then reopen Krnl Executor.

- ❌ **Krnl Executor IPA shows "Untrusted Developer" on iOS**
  Sideloaded apps must be manually trusted before first use on iOS devices.
  **Solution:** Navigate to `Settings > General > Device Management`, tap the developer profile, and select `Trust`.

- ❌ **Scripts execute but produce no visible effect**
  The target game may have server-side validation that blocks certain client-side modifications from rendering.
  **Solution:** Test in a private server or try a different script to isolate whether the issue is with Krnl Executor or the script itself.
